开发者 API 集成文档

如何获取 API Key

  1. 联系管理员，进入后台控制台「API 令牌分发」页面
  2. 管理员填写拥有者标识并点击「签发令牌」
  3. 令牌签发后明文显示一次，需立即复制保存
  4. 此后令牌脱敏展示，若丢失需删除后重新签发

全局规范声明

Base URL：http://<server-ip>:3000/v1

认证 Header：Authorization: Bearer sk-jm-xxxxxxxxxx

请求格式：multipart/form-data — 所有生成接口统一使用 form-data，支持文件上传

任务轮询：GET /v1/tasks/:id — 轮询直到 status 为 success 或 failed。出现 poll_warning 字段时为我方网络抖动，任务未失败，继续等待即可

异步任务响应机制 (必读)

两个生成接口均为异步非阻塞：POST 提交后立即返回任务 ID，后台持续执行。客户端需轮询 GET /v1/tasks/:id 取得最终 URL。
注意：failed 仅在火山服务器明确返回失败时出现；我方网络问题不会将任务标为失败，会通过 poll_warning 字段告知。

① POST 提交 → 立即返回

{
  "id": "clxxxxxxxxxxx",
  "status": "processing",
  "submit_id": "abc123"
}

id 用于后续轮询，submit_id 为火山流水号

② 生成中（正常）

{
  "id": "clxxx",
  "status": "processing"
}

继续轮询，图片建议 5s，视频建议 15s

② 生成中（网络抖动）

{
  "id": "clxxx",
  "status": "processing",
  "poll_warning": "[时间戳] 我方网络
异常，任务仍在火山
排队。错误: ETIMEDOUT"
}

⚠️ 任务未失败！不要重新提交，继续等待即可，网络恢复后自动清除

③ 成功

{
  "id": "clxxx",
  "status": "success",
  "data": [{ "url": "https://..." }]
}

④ 失败（火山明确返回）

{
  "id": "clxxx",
  "status": "failed",
  "error": "火山服务返回失败..."
}

POST {{ config.baseUrl.replace(/\/$/, '') }}/images/generations

model resolution_type ratio 图生图 (images) 备注

📝 Form-Data 参数

  • prompt
    *必填 画面描述提示词
  • model
    模型版本: 5.0, 4.6, 4.5, 4.1, 4.0, 3.1, 3.0
  • ratio
    画幅比例: 16:9, 1:1, 9:16, 4:3, 3:4, 21:9 等
  • resolution_type
    分辨率: 4k / 2k (3.x 系最高 2k)
  • images
    [File Array] 图生图: 上传 1~10 张，仅限 4.0+ 模型

cURL 示例

curl -X POST {{ config.baseUrl.replace(/\/$/, '') }}/images/generations \
  -H "Authorization: Bearer YOUR_KEY" \
  -F "model=5.0" \
  -F "prompt=一只超写实机械猫咪" \
  -F "ratio=16:9" \
  -F "resolution_type=4k"

# 追加 -F "images=@/path/to/ref.jpg"

接口2：视频创作 (POST /v1/videos/generations)

POST {{ config.baseUrl.replace(/\/$/, '') }}/videos/generations

💡 API 路由原则（客户端无需关心）

你只需传 model + prompt + 附件（可选），API 端自动选最优子命令：

  • 3.x 系 无附件 → 文生视频；仅首帧 → 单帧生视频；首帧+尾帧 → 首尾帧生视频
  • seedance2.0 系 无附件 → 文生视频；加图片 → 图生视频/多图叙事；含视频/音频 → 多模态生视频

📊 各模型能力矩阵

model 时长 (s) 最高分辨率 文生视频 首尾帧 多模态·音频 备注
seedance2.0_vip4–15720p✅✅✅ 图+视+音VIP 超极速
seedance2.0fast_vip4–15720p✅✅✅ 图+视+音VIP 极速版
seedance2.04–15720p✅✅✅ 图+视+音标准推荐
seedance2.0fast4–15720p✅✅✅ 图+视+音快速出片
3.5pro4–121080p✅✅ 首·尾帧❌精细高清
3.0pro3–101080p✅✅ 首帧❌高清·单帧
3.0fast3–10720p✅✅ 首帧❌快速
3.03–10720p/1080p✅✅ 首·尾帧❌均衡

📝 通用 Form-Data 参数

  • prompt
    *必填 核心描述提示词
  • model
    模型名称，见上表（支持下划线别名，如 3.5_pro → 3.5pro）
  • duration
    时长整数秒，范围见上表
  • video_resolution
    720p（seedance2.0 系） / 720p 或 1080p（3.5pro / 3.0）/ 1080p（3.0pro）；不传 = 模型默认
  • ratio
    仅 文生视频 和 多模态生视频 生效（16:9 / 9:16 / 1:1 等）；有参考图/帧时 CLI 自动跟首帧比例，可不传
🎞️ 3.x 系模型 — 关键帧文件槽
  • image （第 1 个） — 首帧，决定起始画面
  • image （第 2 个） — 尾帧，决定结束画面（仅 3.5pro / 3.0 支持）
  • 两帧均不传 = 纯文生视频；仅传首帧 = 单帧生视频；两帧都传 = 首尾帧生视频
💥 seedance2.0 系 — 多模态文件槽
  • image — 参考图，最多 9 张；1 张 = 单图生视频，2+ 张 = 多图叙事
  • video — 参考视频，最多 3 个，mp4/mov，单个 2–15 秒 ≤50MB
  • audio — 参考音频，最多 3 首，wav/mp3，单个 2–15 秒 ≤15MB
  • 含视频或音频文件 = 自动升级为多模态生视频；全不传 = 纯文生视频

cURL 示例

# ① 文生视频 (任意模型)
curl -X POST {{ config.baseUrl.replace(/\/$/, '') }}/videos/generations \
  -H "Authorization: Bearer YOUR_KEY" \
  -F "model=seedance2.0" \
  -F "prompt=赛博朋克都市夜景" \
  -F "duration=5" -F "ratio=16:9"

# ② 单帧生视频 (3.x 系)
curl ... -F "model=3.5pro" \
  -F "prompt=镜头慢慢拉远" \
  -F "image=@first.jpg"

# ③ 首尾帧生视频 (3.5pro / 3.0)
curl ... -F "model=3.5pro" \
  -F "prompt=从晨到暮的街道变化" \
  -F "image=@morning.jpg" \
  -F "image=@evening.jpg"

# ④ 多模态 (seedance2.0 系)
